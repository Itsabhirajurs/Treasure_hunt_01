Date: 2026-04-01

====================================================
1) SYSTEM OVERVIEW
====================================================
This platform is a real-time competitive treasure hunt web app.
It supports team onboarding, round-wise clue solving, AI-assisted validation,
progressive hints, scoring, badges, live leaderboard updates, and admin analytics.

Core experience:
- Team registers/logs in.
- Team plays 5 rounds.
- Each submit is evaluated by Gemini + fallback logic.
- Score is calculated via exponential decay.
- Badges and progression are stored in Supabase.
- Leaderboard and admin feed update in real-time.

4) GEMINI API USAGE
====================================================
A) Semantic Answer Validation
- Endpoint: POST /api/submit
- Service: backend/services/gemini_service.py -> validate_answer()
- Prompt asks Gemini to accept synonyms, abbreviations, minor spelling errors.
- Fallback path exists if parsing/model fails.

B) Progressive Hint Generation
- Endpoint: GET /api/hint
- Service: backend/services/gemini_service.py -> generate_hint()
- Hint level changes by hint number:
  1 = cryptic, 2 = helpful, 3 = near-direct.

C) Safety/Fallback Behavior
- If Gemini unavailable, system uses deterministic fallback hints/checks.

====================================================
5) SCORING LOGIC
====================================================
Implemented in: backend/services/scoring_service.py

Formula:
raw_score = base * exp(-0.002 * time_elapsed) * hint_penalty * round_multiplier

Where:
- base = 1000
- hint_penalty = max(0.1, 1 - 0.20*hints_used)
- round_multiplier = 1 + (round_number - 1) * 0.3
- final score = max(10, round(raw_score))

Effect:
- Faster teams score higher.
- Hints reduce score.
- Higher rounds are worth more.

====================================================
6) ANOMALY DETECTION LOGIC
====================================================
Implemented in: backend/services/anomaly_service.py

Expected ranges:
- Round 1: 20 to 120 sec
- Round 2: 30 to 180 sec
- Round 3: 45 to 240 sec
- Round 4: 60 to 300 sec
- Round 5: 90 to 400 sec

Rule:
- If time < 30% of round minimum, flag suspicious.

9) RUNTIME CHECKLIST (SMOOTH OPERATION)
====================================================
1. Backend .env must have valid:
   - SUPABASE_URL
   - SUPABASE_KEY (service role)
   - GEMINI_API_KEY
   - FRONTEND_ORIGIN

2. Frontend .env must have valid:
   - VITE_SUPABASE_URL
   - VITE_SUPABASE_ANON_KEY
   - VITE_BACKEND_URL=http://localhost:5000

3. Supabase SQL setup must be executed:
   - teams, submissions, clues tables
   - realtime publication for teams + submissions

4. Start backend:
   cd backend
   py app.py

5. Start frontend:
   cd frontend
   npm run dev

6. Health checks:
   - GET /api/health should return ok
   - GET /api/admin/schema-status should return ready=true
